Below is the exact and detailed UX-10 patch pack.

UX-10 goal

Stabilize the result surface so it behaves like a product, not a race between async workers and layout switching.

This phase addresses exactly these four issues:

photo grid refresh immediately after async load completes
project-id propagation before layout reload
avoiding project_id=None reload races on layout switch
making Google layout refresh from store events rather than side-channel timing

This is a timing, ownership, and refresh-coordination phase.
It is not a new UX section phase.

Design rules for UX-10
Rule 1, layout reloads must never guess project state

A layout must not ask, “what is the current project right now?” at reload time if that state is still in flight.

It must receive a resolved project id or refuse to reload.

Rule 2, async loaders must signal completion once, centrally

Do not let:

grid worker
sidebar
layout
search shell

all guess when loading is “done”.

There must be one completion signal, then one coordinated refresh path.

Rule 3, search-state-driven layouts beat side-channel refreshes

Google layout should react to:

SearchStateStore
resolved result paths
explicit project id

not ad hoc timer timing.

Rule 4, project_id=None is onboarding only

It must never be treated as a transient “maybe live project” reload state.

If project is temporarily unknown during switching, layouts should wait, not reload.

UX-10 implementation overview

This patch has six parts:

SearchStateStore stabilization fields
SearchController project-resolution helpers
MainWindow project-switch transaction guard
LayoutManager stable layout-attach / reload rules
GooglePhotosLayout refresh from store, not timers
result-grid completion hook and centralized UI refresh mediator

def _safe_reload_current_layout(self):
    try:
        if self._project_switch_in_progress:
            self._pending_layout_reload = True
            return

        pid = getattr(self, "current_project_id", None)
        if pid is None:

5.3 Replace _on_search_state_changed() with generation-aware version

Replace with:

def _on_search_state_changed(self, state):
    try:
        if not hasattr(self, "empty_state"):
            return

        if getattr(state, "onboarding_mode", False):
            self._show_empty_state("no_project", getattr(state, "warnings", []))
            return

        if not getattr(state, "active_project_id_resolved", False):
            return

        if getattr(state, "layout_reload_pending", False):
            return

        displayed_paths = list(getattr(state, "displayed_result_paths", []) or [])
        result_paths = list(getattr(state, "result_paths", []) or [])
        search_in_progress = bool(getattr(state, "search_in_progress", False))
        result_surface_busy = bool(getattr(state, "result_surface_busy", False))

        # Preserve visible results while searching / busy
        if search_in_progress or result_surface_busy:
            if displayed_paths:
                self._last_store_result_paths = displayed_paths
                self._show_results_surface()
            else:
                self._show_empty_state("loading", getattr(state, "warnings", []))
            return

        # Indexing state with no results
        if getattr(state, "indexing_in_progress", False) and not result_paths:
            self._show_empty_state("indexing", getattr(state, "warnings", []))
            return

        # Embeddings warning with no results
        if not getattr(state, "embeddings_ready", True):
            reason = getattr(state, "empty_state_reason", None)
            if reason == "embeddings_missing" and not result_paths:
                self._show_empty_state("embeddings_missing", getattr(state, "warnings", []))
                return

        if result_paths:
            self._last_store_result_paths = result_paths
            self._show_results_surface()
            self._refresh_timeline_from_store_paths(result_paths)
            return

        self._show_empty_state(getattr(state, "empty_state_reason", None) or "no_results",
                               getattr(state, "warnings", []))

    except Exception as e:
        print(f"[GooglePhotosLayout] _on_search_state_changed error: {e}")
5.4 Add _refresh_timeline_from_store_paths(paths)

Add:

def _refresh_timeline_from_store_paths(self, paths):
    try:
        if not hasattr(self, "timeline"):
            return

        if hasattr(self.timeline, "set_photo_paths"):
            self.timeline.set_photo_paths(paths)
        elif hasattr(self.timeline, "load_paths"):
            self.timeline.load_paths(paths)
        elif hasattr(self.timeline, "display_thumbnails"):
            self.timeline.photo_paths = list(paths)
            self.timeline.display_thumbnails()

    except Exception as e:
        print(f"[GooglePhotosLayout] _refresh_timeline_from_store_paths failed: {e}")

This is the critical step that stops Google layout from waiting on side-channel reload timing.
